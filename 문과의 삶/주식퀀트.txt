1) 목적 규정: 분류·회귀·랭킹
A. 분류 (Classification)
출력: “라벨”이오. (예/아니오, A/B/C 같은 범주)
예시
스팸메일인가?
대출 연체할 고객인가?
공시가 ‘악재’로 반응할 가능성이 큰가/작은가?
전형적 지표: Accuracy, Precision/Recall, F1, ROC-AUC
(특히 금융·리스크는 Recall/Precision 균형이 중요하오.)
퀀트 맥락
“내일 수익률이 +냐 -냐” 같은 방향성 예측을 분류로도 하오.
다만 시장은 교묘해서, 분류 정확도 55%여도 돈이 되기도 하고 60%여도 돈이 안 되기도 하오(비용·슬리피지·포지션 크기 때문이오).

B. 회귀 (Regression)
출력: “숫자”요. 연속값이오.
예시
다음 분기 매출/수요 예측
변동성(σ) 예측
다음날 기대수익률(연속값) 예측
전형적 지표: MAE, RMSE, R², MAPE(업종 따라)
퀀트 맥락
기대수익률, 변동성, 베타, 스프레드 등 연속량 추정에 자주 쓰오.
단, 금융은 노이즈가 두꺼워서 회귀는 “맞춘다”보다 “덜 틀린다”가 목표요.

C. 랭킹 (Ranking / Learning-to-Rank)
출력: “순위”요. 절대값보다 상대서열을 맞추는 데 집중하오.
예시
종목 2,000개 중 상위 50개 뽑기
추천 시스템(유저별 Top-N)
신용/리스크 우선순위 큐 만들기
전형적 지표: NDCG, MAP, HitRate@K, Spearman, IC(Information Coefficient)

퀀트 맥락
실전에서 제일 현실적인 형태가 많소.
“수익률을 정확히 맞춘다”는 허상이기 쉬우나,
“상대적으로 더 나을 애들을 위로 올린다”는 가능성이 남소.
업계에서 “ML 한다”라고 하면 통상 뭘 말하나
분야별로 관용이 다르오:
데이터/서비스 업계(추천·검색): 거의 항상 랭킹 또는 분류
핀테크/리스크(사기·연체): 거의 항상 분류
수요·가격·예측(공급망/리테일): 주로 회귀
퀀트/자산운용:
“리턴 예측”을 말로는 회귀라 해도, 실전 운용은 **랭킹(상대강도/스코어링)**으로 귀결되는 경우가 많소.
이벤트(공시/뉴스) 반응은 분류로 시작하기도 하오.

1단계: 문제정의 (여기서 절반이 끝나오)
반드시 문장으로 고정하시오.
입력 X: 우리가 미리 알 수 있는 정보(특징)
출력 y: 맞추려는 것(라벨)
목적: 분류/회귀/랭킹 중 하나
평가지표: 무엇을 “잘했다”고 볼 것인가
제약: 비용(거래비용), 지연(데이터가 언제 생기는지), 누수 금지
예시(뉴스/공시 기반):
분류: “공시 후 1일 누적수익률이 -1% 이하로 떨어지면 1, 아니면 0”
회귀: “공시 후 1일 누적수익률(연속값)”
랭킹: “당일 공시 종목들을 다음날 수익률 기준으로 상위로 정렬”

2단계: 데이터 (X,y 만들기)
핵심 원칙
ML은 데이터를 먹고 사는 짐승이오.
특히 금융은 라벨이 더 중요하오. 라벨 설계가 구리면 모델은 전부 구리오.
데이터 형태(표준)
행(row) = 관측치(예: “종목-날짜”, “뉴스-시간”)
열(column) = 특징(feature)
y = 목표값
예시 스키마:
date, ticker
특징: sentiment_score, article_count, volatility_5d, return_1d_lag, ...
라벨: y_return_1d 또는 y_down1pct 등

3단계: 분할 (금융에서 가장 많이 죽는 함정)
랜덤 분할로 train/test 섞으면, 미래 정보가 과거로 새어 들어가오(누수).
시계열: 과거로 학습 → 미래로 테스트
예: 2018–2022 train, 2023 valid, 2024 test

4단계: 베이스라인 (무조건 세워라)
“아무거나 돌린 XGBoost”는 실력처럼 보이나, 대부분은 착각이오.
분류 베이스라인: “항상 0 예측”
회귀 베이스라인: “항상 평균 예측”
랭킹 베이스라인: “간단한 팩터 하나로 정렬(예: 모멘텀)”
베이스라인을 못 이기면, ML을 할 자격이 없소. (강한 말이지만 진실이오.)

5단계: 첫 모델 (scikit-learn로 시작)
분류 예시(로지스틱 회귀)
import pandas as pd
from sklearn.model_selection import TimeSeriesSplit
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_auc_score

df = pd.read_csv("data.csv")

features = ["sentiment_score", "article_count", "volatility_5d"]
X = df[features].values
y = df["y_down1pct"].values  # 0/1

tscv = TimeSeriesSplit(n_splits=5)
aucs = []

for train_idx, test_idx in tscv.split(X):
    X_train, X_test = X[train_idx], X[test_idx]
    y_train, y_test = y[train_idx], y[test_idx]

    model = LogisticRegression(max_iter=2000)
    model.fit(X_train, y_train)
    prob = model.predict_proba(X_test)[:, 1]
    aucs.append(roc_auc_score(y_test, prob))

print("AUC mean:", sum(aucs)/len(aucs))

회귀 예시(릿지 회귀)
from sklearn.linear_model import Ridge
from sklearn.metrics import mean_absolute_error

y = df["y_return_1d"].values

maes = []
for train_idx, test_idx in tscv.split(X):
    model = Ridge(alpha=1.0)
    model.fit(X[train_idx], y[train_idx])
    pred = model.predict(X[test_idx])
    maes.append(mean_absolute_error(y[test_idx], pred))
print("MAE mean:", sum(maes)/len(maes))

6단계: 특징공학(Feature Engineering)
여기서 승부가 나오오.모델을 바꾸는 것보다, 특징을 바꾸는 게 더 세다는 걸 곧 알게 될 것이오.
금융/뉴스에서 자주 쓰는 것들:
변화율: 감성 점수의 절대값보다 전일 대비 변화
빈도: 기사 수, 특정 키워드 출현수
변동성/거래대금 같은 시장상태(레짐)
이벤트 윈도우(공시 전후 1~5일)
7단계: 튜닝과 과적합 방지
튜닝은 “성능 올리기”가 아니라 “폭발 안 하게 하기”요.
방법:
규제(Regularization)
단순 모델 우선
피처 수 줄이기
검증 기간을 여러 레짐으로 나누기(불장/약장)
8단계: 해석 (왜 먹히는가)
금융은 “맞췄다”보다 “왜 맞췄나”가 생존을 가르오.
최소한:
피처 중요도(트리)
계수(선형)
구간별 성능(레짐별)
9단계: 퀀트라면, 반드시 백테스트로 끝내라
모델 지표(AUC, MAE)가 좋아도,
거래비용
슬리피지
포지션 제한
을 넣는 순간 수익은 증발하오.
